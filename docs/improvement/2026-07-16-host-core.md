## Proposals

Findings were checked against callers and tests. These are intentionally not applied because they affect cancellation, error observability, security policy, or error identity.

### High

- **Security — `core/internal/forge/forge.go:112,148`** — errors expose the complete combined output from `gh`/`glab`; CLI output can contain remote URLs, local paths, or authentication-related diagnostics. Suggestion: return a sanitized stable error and log/redact diagnostics through a controlled logger.
- **Security — `core/service/service.go:646-647`** — `runCapture` returns arbitrary combined tool output to callers; build/pull tooling can echo environment-derived or credential-bearing diagnostics. Suggestion: redact sensitive patterns and cap/partition diagnostic output before surfacing it.

### Medium

- **Security — `core/internal/worktree/worktree.go:72-75,129-146,190-192`** — git combined stderr is embedded verbatim in returned errors. Suggestion: keep a bounded/sanitized diagnostic, avoiding raw command output in user-facing error strings.
- **Performance / bug review — `manage/manage.go:152-159`** — `run` uses `exec.Command` without a context, so `go get`, `go tidy`, or `go build` cannot be cancelled and may leave the CLI wedged. Suggestion: thread a command context from the command entry points and use `exec.CommandContext`; this requires API/control-flow decisions, so it is not a safe-fix.
- **Code quality / bug review — `core/internal/manager/session.go:143,167,193,205,212,219,222`** — propagated errors use `%v`, losing `errors.Is`/`errors.As` identity. Suggestion: use `%w` where callers can act on the underlying error and add wrapping assertions; changing error identity is observable API behavior.
- **Performance — `core/internal/manager/gateways.go:20`** — `seen := map[string]bool{}` is allocated even for the empty/default path. Suggestion: fast-path empty input or use a smaller representation if profiling shows this parser is hot; not applied because it adds branching and is not evidenced as a bottleneck.

### Low

- **Doc drift — `core/service/service.go:10-17`** — package documentation describes the installed service as `dctl`, while the host CLI and generated commands are `herrscher`/`herrscherd` in several paths. Suggestion: reconcile the terminology after confirming the intended compatibility names; not changed because it may be deliberate compatibility wording.
- **Code quality — `manage/manage.go:156`** — command display builds a temporary slice with `append` on every invocation. Suggestion: preallocate or use a small formatter; negligible cost and not applied as it would be cosmetic without a measurable hot path.

## Skipped findings

- **`manage/manage.go:153` — `exec.Command` vs `exec.CommandContext`:** confirmed, but skipped as a proposal because introducing cancellation requires changing the command API/lifetime and therefore behavior.
- **`core/internal/forge/forge.go` and `core/internal/worktree/worktree.go` raw command output:** confirmed, but skipped as proposals because sanitizing output changes diagnostics and potentially compatibility with callers/tests.
- **`core/internal/manager/session.go` `%v` → `%w`:** confirmed in selected propagation paths, but skipped as a proposal because error identity is observable behavior.
- **Potential concurrency issues in host loops and terminal gateway:** investigated; no finding retained without a reproducible race or invariant violation in the current tests/code. No speculative fix applied.
- **External dependencies:** not audited, per scope.
